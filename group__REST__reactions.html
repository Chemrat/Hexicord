Add reaction to message.

This method requires READ_MESSAGE_HISTORY permission and, additionally, ADD_REACTIONS permissions if nobody else reacted using this emoji.

Throws RESTError if invalid IDs specified or current user don't have required permissions. Also may throw boost::system::system_error if client fails to connect to REST API server.

Remove reaction from message.

By default this method removes current user's reaction, pass userId to remove other user's reactions, however this requires MANAGE_MESSAGES permission.

Throws RESTError if invalid IDs specified or current user don't have required permissions. Also may throw boost::system::system_error if client fails to connect to REST API server.
